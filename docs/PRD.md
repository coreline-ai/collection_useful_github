# PRD: GitHub 카드 대시보드

## 1. 제품 개요
GitHub 저장소 URL을 입력하면 저장소를 카드 형태로 수집/관리하고, 상세 팝업에서 README/활동/메모를 확인할 수 있는 웹앱이다.

## 2. 목표
- 저장소 핵심 정보를 한눈에 비교 가능하게 제공
- 상세 조회를 캐시 기반으로 최적화해 GitHub API 호출량 절감
- 사용자가 필요할 때만 수동 번역을 실행해 비용/지연 최소화

## 3. 사용자
- 여러 오픈소스 저장소를 조사/정리하는 개발자
- 개인 아이디어/리서치 메모를 저장소 단위로 남기고 싶은 사용자

## 4. 핵심 기능
### 4.1 저장소 등록
- 입력 포맷: `owner/repo`, `https://github.com/owner/repo`, `github.com/owner/repo`
- 유효성 검증 실패 시 에러 표시
- 중복 저장소 등록 차단

### 4.2 카드 리스트
- 4열 그리드(데스크톱), 반응형 열 축소(태블릿/모바일)
- 카드 정보: 저장소명, owner, summary(최대 3줄), language, stars, forks, 업데이트일, 링크
- 카드 액션: 상세 보기, 삭제
- 페이지네이션: 페이지당 12개, 숫자 + 이전/다음

### 4.3 상세 팝업
- 탭: `개요`, `README`, `Activity`
- 개요: 핵심 메타 + summary + 링크
- README: Markdown 렌더 (sanitize + GitHub-like 스타일)
- Activity: 커밋/이슈/PR 활동 목록
- 메모: 입력/누적 표시, 최대 500자

### 4.4 수동 번역
- 자동 번역 없음
- 각 탭에 번역 버튼 제공
  - 개요 번역
  - README 번역
  - Activity 번역
- 번역 후 `원문 보기` 토글 가능

### 4.5 캐시 전략
- 상세 데이터는 repo별 localStorage 캐시 사용
- 최초 조회 시만 원격 상세 호출
- `업데이트 확인` 버튼으로 최신 커밋 SHA 비교
- SHA 변경 시 `최신 데이터 불러오기` 버튼 노출
- 캐시 TTL: 24시간

### 4.6 블랙테마
- 테마 모드: `light | dark`
- 동작 정책:
  - 저장값 있으면 저장값 우선
  - 저장값 없으면 OS `prefers-color-scheme` 자동 감지
- 상단 우측 토글로 즉시 전환
- 전환 결과는 localStorage에 저장되어 새로고침 후 유지
- 다크 테마는 순수 블랙(`#000`) 기반으로 적용

## 5. 비기능 요구사항
- 에러 메시지는 사용자 행동 유도형으로 제공(예: 토큰 설정 안내)
- README 렌더는 XSS 방지(sanitize)
- 로컬 저장소 사용으로 새로고침 후 상태 유지

## 6. 성공 기준
- 동일 저장소 상세 팝업 재진입 시 원격 재호출 없이 캐시 로드
- SHA 변경이 없을 때 상세 업데이트 호출이 발생하지 않음
- 카드 리스트에서 summary가 3줄 이내로 안정 표시
- 주요 플로우(등록/삭제/상세/메모/번역) 테스트 통과
- 저장값 없는 첫 진입에서 OS 다크 모드 감지가 정상 적용됨
- 테마 토글 후 새로고침 시 마지막 선택 테마가 유지됨

## 7. 제외 범위
- 사용자 계정/로그인
- 서버 DB 및 다중 기기 동기화
- 조직/유저 전체 저장소 일괄 수집
